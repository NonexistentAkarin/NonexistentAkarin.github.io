从现在开始使用 Firebase Cloud Messaging (FCM) 推送通知

Firebase Cloud Messaging (FCM) 是一种跨平台消息传递解决方案，我们可以使用它在Android系统上推送消息和通知。

相比国内各种各样的第三方推送服务，如小米推送、腾讯信鸽推送、极光推送等，FCM的最大优势是他是系统级别的推送服务，可以获得像iOS那样任何时候都可以推送给用户的效果，而不必担心推送的后台服务被杀的情况。因为是Android系统级别的服务，FCM不需要像其他第三方推送服务那样，在后台维护一个Service，维持应用与服务端之间的TCP长连接。

这里简单讲述如何把FCM集成到我们的应用中。

在网页端申请并配置使用Firebase

1.首先来到Firebase网站：https://firebase.google.com

登陆我们自己的Google账户，点击右上角的”转到控制台”。

2.进入到Firebase的控制台界面。

3.点击 “添加项目”。

4.创建完项目后会显示”欢迎使用Firebase”的界面。

这里我们选择”将Firebase添加到您的Android应用”。

5.输入”Android软件包名称”和”应用别名”，点击”注册应用”。

6.点击”下载google-services.json”，并把下载的google-services.json放入我们应用项目的app文件夹中。

7.在build.gradle中添加Firebase SDK依赖。

  • 在项目级 build.gradle（/build.gradle）的 dependencies{} 中添加：
    classpath 'com.google.gms:google-services:3.1.0'
  • 在应用级 build.gradle（//build.gradle）的最下面添加：
    apply plugin: 'com.google.gms.google-services'
  • 最后点击 IDE 中显示的栏中的”立即同步”。

点击”完成”。

8.然后就可以在概览中看到刚刚创建的项目。

9.点击左侧的”Notifications”，进入通知界面，再点击”查看文档”，可以查看实现FCM推送的实现路径。

在网页下方有Google提供的Demo地址：https://github.com/firebase/quickstart-android/tree/master/messaging

在Android项目中配置使用Firebase

1.在应用级 build.gradle（//build.gradle）里面添加相关依赖。

compile 'com.google.firebase:firebase-core:11.0.1'
compile 'com.google.firebase:firebase-messaging:11.0.1'
compile 'com.firebase:firebase-jobdispatcher:0.6.0'

2.添加相关代码。

  • 添加三个Service文件到项目中（Google的Demo中有提供）
    • MyFirebaseInstanceIDService
    • MyFirebaseMessagingService
    • MyJobService
  • 修改AndroidManifest
    • 添加联网权限
      <uses-permission android:name="android.permission.INTERNET" />
    • 注册这三个Service

使用Firebase推送通知

1.在Firebase网页端通知界面，点击发送您的第一条消息，”目标”选择”单个设备”，提示需要FCM注册令牌。

2.在Android项目的MyFirebaseInstanceIDService中，可以通过Log的形式，将FCM注册令牌打印到日志。

3.将日志中打印出的token作为FCM注册令牌填入，点击”发送消息”。

跳转到推送”已完成”的页面。

4.于是我们在手机端也已经收到了通知。

这说明我们已经使用FCM推送成功。
